# Book Library API

A small, maintainable REST API for managing a personal book collection. The service is built with FastAPI, SQLAlchemy, and SQLite, and includes request validation plus interactive OpenAPI documentation.

## Features

- Create, list, retrieve, partially update, and delete books.
- Pagination with `skip` and `limit` query parameters.
- Filter by genre.
- Sort by rating or date added.
- Validation for title, author, genre, rating, and notes.
- SQLite by default, configurable through `DATABASE_URL`.
- Swagger UI and ReDoc generated by FastAPI.
- Streamlit web interface for browsing and managing books.

## Technology stack

- Python 3.11+
- FastAPI 0.104.1
- Uvicorn 0.24.0
- SQLAlchemy 2.0.23
- Pydantic 2.5.0
- python-dotenv 1.0.0
- SQLite for local development

## Installation

From the project root, create and activate a virtual environment:

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

On macOS or Linux, use:

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Create a `.env` file in the project root if one does not exist:

```env
DATABASE_URL=sqlite:///./media_library.db
```

For deployment, use a hosted PostgreSQL connection string from Neon or another database provider instead of SQLite. Do not place credentials or production secrets in a committed `.env` file.

## Running the API

Start the development server from the project root:

```bash
uvicorn app.main:app --reload
```

The service is available at `http://127.0.0.1:8000`.

- Swagger UI: `http://127.0.0.1:8000/api/docs`
- ReDoc: `http://127.0.0.1:8000/api/redoc`
- Health check: `http://127.0.0.1:8000/`

### Live API documentation

The deployed Swagger UI is available at [Book Library API Docs](https://book-library-api-three.vercel.app/api/docs#/).

## Running the Streamlit UI

Start the FastAPI server first, then open a second terminal with the virtual environment activated:

```bash
streamlit run streamlit_app.py
```

The UI opens at `http://localhost:8501` and connects to `http://127.0.0.1:8000/api/v1` by default. To connect to another API deployment, set `API_BASE_URL` in `.env`:

```env
API_BASE_URL=https://your-api.example.com/api/v1
```

The UI supports listing, filtering, sorting, adding, editing, and deleting books. It communicates with the API over HTTP and does not access the database directly.

## API usage

### Create a book

```bash
curl -X POST "http://127.0.0.1:8000/api/v1/books/" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "The Great Gatsby",
    "author": "F. Scott Fitzgerald",
    "genre": "fiction",
    "rating": 4.5,
    "notes": "A classic American novel"
  }'
```

### List and filter books

```bash
curl "http://127.0.0.1:8000/api/v1/books/?skip=0&limit=20&genre=fiction&sort_by=rating"
```

### Retrieve, update, or delete a book

```bash
curl "http://127.0.0.1:8000/api/v1/books/1"

curl -X PATCH "http://127.0.0.1:8000/api/v1/books/1" \
  -H "Content-Type: application/json" \
  -d '{"rating": 4.8, "notes": "Updated note"}'

curl -X DELETE "http://127.0.0.1:8000/api/v1/books/1"
```

The supported genres are `fiction`, `non_fiction`, `mystery`, `sci_fi`, `fantasy`, `biography`, `history`, `science`, and `other`. Ratings must be between `0` and `5`; list limits must be between `1` and `100`.

## Architecture

Requests from the Streamlit UI are sent to the FastAPI API. API requests enter the FastAPI router, are validated by Pydantic schemas, pass to the book service, and are persisted through SQLAlchemy. Database sessions are provided as request dependencies and closed after each request. See [docs/SDLC.md](docs/SDLC.md) for the full SDLC, data model, validation strategy, and deployment roadmap.

## Testing and quality

Automated tests, CI workflows, migration tooling, and lint configuration are not yet included in this repository. Before production release, add pytest coverage for schemas, services, endpoint success paths, validation failures, missing resources, filtering, sorting, and pagination. The recommended quality commands are:

```bash
black app tests
isort app tests
flake8 app tests
pytest
```

## Production considerations

The default SQLite setup is intended for local development and small workloads. A production deployment should use a managed database, Alembic migrations, restricted CORS origins, authentication, HTTPS, structured logging, backups, monitoring, and secret-manager-backed configuration.
